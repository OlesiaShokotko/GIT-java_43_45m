gir status отображает состояние рабочего каталога и раздела проиндексированных файлов. С ее помощью можно проверить индексацию изменений и увидеть файлы, которые не отслеживаются Git.
git log отображает все изменения(commit), происходившие с нашим репозиторием в обратном порядке.
git remote -v позволяет убедиться, что репозиторий, склоннированный и сохраненный локально на наше устройство, действительно связан с репозиторием, созданным на GitHub.

1. Предварительно должен быть создан репозиторий на github.com, а также должен быть указан public ssh key. Далее необходимо нажать на "Code", в выпадающем списке выбрать SSH, скопировать. В командной строке GitBush ввести команду git config --global -l, убедиться, что имя и электронная почта указаны. И при помощи команды git clone и скопированного SSH клонировать репозиторий локально.
2. Заходим в локальный каталог, который хотим отправить. Используем команду git init, командой ls -la проверяем, был ли создан файл .git, командой git status проверяем наличие неотслеживаемых файлов, далее вводим команду git add ., далee git commit -m " ", далее создаем репозиторий на github.com, в GitBush вводим команду git remote add origin "SSH", далее git branch -M main и git push -u origin main.
3. Для получения изменений с github.com на локальное устройство необходимо воспользоваться командой git pull.
